# TRACE End-to-End QA Automation ‚Äî Plan & Progress Tracker

**Created:** 2026-06-03 ¬∑ **Owner:** Eng ¬∑ **Status:** In progress

## Context
The platform had **no UI/E2E coverage** ‚Äî CI only runs API unit/integration tests (vitest) against Postgres/Redis. After the workshop changes (supplier seller flow, photo-gated listings, logged-out sign-up CTA, demo anchor simulation, mobile nav), nothing automatically exercises those journeys in a browser. This effort adds a Playwright E2E layer that protects the 8 June 2026 workshop journeys against regressions, plus a later AI exploratory QA agent.

## Scope decisions (confirmed)
- **Both**: deterministic Playwright suite now; AI exploratory agent later.
- **Targets**: ephemeral seeded stack in CI (primary gate) **and** post-deploy smoke against the live domain.
- **Coverage**: workshop-critical journeys + mobile (375px) + visual-regression matrix.

## Design decisions
- `@playwright/test` lives in **`packages/e2e`** (`@trace/e2e`), kept out of `web`.
- **Determinism via `DEMO_SIMULATE_ANCHOR=true`** in the test stack ‚Üí the trust layer is instant and never flaps, so blockchain can't cause flakiness.
- **Auth via `storageState`**: `global-setup` logs each seeded role in through the API and saves a state file (the app keeps the JWT in `localStorage` + a `trace_auth` cookie); specs opt in with `test.use({ storageState })`.
- **Test data = seeders**: `seed` (hub staff + sample passports) + `seed:workshop` (supplier accounts `ada.lovelace@‚Ä¶`, `grace.hopper@‚Ä¶` / `TraceWorkshop2026!`).
- **Selectors**: prefer user-facing (`getByRole`/`getByText`); add a few `data-testid`s for genuinely ambiguous/structural elements (listing card, trust badge, sign-up CTA, mobile nav toggle, list-for-sale).
- **Visual baselines generated in the pinned Playwright container** (`mcr.microsoft.com/playwright`) so snapshots match CI byte-for-byte.
- The Playwright script is named **`e2e`** (not `test`) so `turbo test` doesn't try to run browsers.

## Phases

### Phase 0 ‚Äî Foundation
`packages/e2e/`: `package.json`, `tsconfig.json`, `playwright.config.ts` (projects: `desktop-chromium`, `mobile-375`, `tablet`), `global-setup.ts` (role storageStates), `fixtures/test-helpers.ts` (`noHorizontalOverflow`, account map, state paths), `README.md`, `.gitignore`. Root `e2e*` scripts. `E2E_BASE_URL` / `E2E_API_URL` config.

### Phase 1 ‚Äî Critical journey specs
`public-marketplace`, `supplier-journey`, `photo-guard`, `buyer-journey`, `roles` (see tracker).

### Phase 2 ‚Äî Mobile + visual matrix
Critical specs under `mobile-375` (hamburger nav, tappable CTAs, no horizontal overflow); `toHaveScreenshot()` baselines for marketplace, listing detail, passport detail, login, register, wizard, my-listings (desktop + mobile).

### Phase 3 ‚Äî CI gate (ephemeral stack)
`.github/workflows/e2e.yml`: services postgres + redis + **minio** (photo upload) + `DEMO_SIMULATE_ANCHOR=true`; build ‚Üí migrate ‚Üí `seed` + `seed:workshop` ‚Üí start api & web ‚Üí `playwright install --with-deps chromium` ‚Üí `pnpm e2e` ‚Üí upload HTML report + traces.

### Phase 4 ‚Äî Deployed-domain smoke
`@smoke`-tagged read-mostly subset run with `E2E_BASE_URL=https://trace.adventurous.systems` post-deploy; dedicated `smoke-supplier` account; created data cleaned up; alerts, doesn't block.

### Phase 5 ‚Äî AI exploratory QA agent (after deadline)
Opt-in Claude + Playwright MCP that explores journeys/edge cases (a11y, odd inputs, responsive) and emits a markdown findings report; runnable via `/loop` or a scheduled routine. Deterministic suite stays the gate.
